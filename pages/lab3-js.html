TNMK30 Laboration 3: JavaScript

I den här laborationen ska vi undersöka hur vi kan koda in beteende för en webbsida med scriptspråket JavaScript. Vi har plockat ihop en liten crash course i JavaScript så att vi kan känna oss redo innan vi kör igång på riktigt.

Föruppgift (behöver ej redovisas)

Det första vi behöver göra är att, precis som med css, länka in vår js-fil i html-dokumentet. Vi vill oftast ladda in script sist. Så här kan det se ut: 

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <title>TNMK30 Lab 3</title>
</head>
<body>
    <!-- kanske lite html -->
    <script src="minjavascriptfil.js"></script>
</body>
</html>

Att koda JavaScript är ganska likt c++ och det första vi lärde oss i c++ var antagligen att skriva ut saker i kommandotolken med cout. De allra flesta webbläsare har en egen kommandotolk, eller console, som vi kan skriva till. Denna finns under tools (chrome) eller tools > web developer (firefox). Öppna den, så kan vi se vad som händer i vår JavaScript.

Föruppgiften går ut på att översätta ett program skrivet i c++ till JavaScript. Programmet sorterar en array med bubblesort (som vi känner igen från TND012). För att lyckas med en översättning behöver vi lite syntax från JavaScript att jobba med. Testa dem gärna separat och se vad de gör:

JavaScript Motsvarar (ungefär) c++
Skriva ut saker till konsollen console.log("Hello World"); cout << "Hello World";
Deklarera variabel var myVar = 1; myVar = "hej"; int myInt = 1; string myString = "hej";

Lite om HTML5 Canvas

I HTML5 finns ett nytt element som heter <canvas>, som kan användas för att skapa interaktiv grafik med hjälp av JavaScript. Vi har gjort i ordning några exempel som visar hur man kan rita några enkla former, men också hur man kan få något att röra på sig.

Uppgift

För godkänt ska din webbapplikation:

  • Använda en loop för att rita upp flera objekt, eller rita en spännande form.
  • Innehålla en animation eller reagera på att användaren gör något (till exempel klickar på en knapp eller rör på musen).
  • Vara bra strukturerad med hjälp av funktioner.

Idétorka?

Ni är fria att göra vad ni vill, men här är några exempel som skulle kunna vara av rimlig omfattning:

  • Ett program, där man får välja dimensionerna hos en legobit som sedan ritas upp.
  • Några bollar som studsar runt.
  • En gubbe som nickar eller skakar på huvudet.
  • Snöflingor som faller.
